# Specs

Specs live here so product intent, app behavior, and code changes stay connected.

Start with `product.md` for the application-wide picture. Use `templates/feature-spec.md` for new features, then place the filled spec in `features/`.

Run:

```bash
npm run spec:check
```

The checker is intentionally simple. It makes sure feature specs keep the core sections that make implementation and review easier.
